ТЕХНИЧЕСКИЙ ПАСПОРТ ПРОЕКТА: SDR Frequency Manager & AR Satellite Tracker (PWA)
1. Общее описание приложения
Проект представляет собой автономное веб-приложение (Progressive Web App - PWA) для радиолюбителей и пользователей SDR (Software-Defined Radio). Приложение предназначено для удобного просмотра, фильтрации и поиска радиочастот из базы данных (XML), а также для визуального отслеживания положения спутников на реальном небе с помощью технологий дополненной реальности (AR) в браузере мобильного устройства. Приложение ориентировано на работу в условиях отсутствия интернета (оффлайн-режим).
2. Инструментарий разработчика
Среда разработки: Cursor IDE.
Инструменты ИИ: Встроенный функционал Cursor Composer (на базе Claude 3.5 Sonnet / GPT-4o или аналогичных моделей).
Языки: Чистый HTML5, CSS3, JavaScript (Vanilla JS, без тяжелых фреймворков вроде React/Vue для максимальной легкости).
3. Архитектура и Хостинг
Платформа хостинга: GitHub Pages (бесплатный хостинг статических сайтов).
Тип приложения: Статическое Single Page Application (SPA), обернутое в PWA. Backend-сервер отсутствует, вся логика выполняется на стороне клиента (в браузере).
Безопасность: GitHub Pages автоматически предоставляет HTTPS-соединение, что является критически важным и обязательным требованием браузеров (iOS Safari, Android Chrome) для доступа к камере, GPS и гироскопу смартфона.
CORS (Cross-Origin Resource Sharing): Поскольку исполняемые файлы (HTML/JS) и файлы данных (XML/TLE) находятся в одном репозитории на GitHub, проблемы с блокировкой ресурсов политиками CORS исключены.
4. Источники и структуры данных
База частот (Frequencies.xml): Статический XML-файл, выгружаемый из программы SDR#. Содержит теги MemoryEntry с параметрами: <Name>, <Frequency>, <GroupName>, <DetectorType>, <FilterBandwidth>.
NORAD ID: Уникальный идентификатор спутника, который программно извлекается из тега <Name> (заключен в квадратные скобки, например, [28117]).
Орбитальные данные (TLE): Текстовый файл с актуальными кеплеровыми элементами орбит спутников, необходимый для математического расчета их положения.
5. Основные функциональные модули
Модуль 1: Менеджер частот (Таблица и UI)
Парсинг: JavaScript (DOMParser) считывает Frequencies.xml и преобразует его в HTML-таблицу.
Вкладки (Группы): Автоматическая генерация кнопок-вкладок на основе уникальных значений из тега <GroupName> (например, SATCOM, MERIDIAN). При клике таблица фильтруется.
Живой поиск: Текстовое поле input, которое при вводе каждого символа мгновенно фильтрует строки таблицы по совпадениям в названиях или частотах.
Модуль 2: AR-Трекер спутников (Дополненная реальность)
Триггер: При клике на строку таблицы приложение извлекает NORAD ID и переходит в режим трекинга этого конкретного спутника.
Камера: Использование navigator.mediaDevices.getUserMedia для вывода видеопотока с задней камеры смартфона в фоновый тег <video>.
Сенсоры:
Использование navigator.geolocation для получения GPS-координат пользователя (работает оффлайн).
Использование DeviceOrientationEvent (азимут/компас и наклон/гироскоп) для определения направления камеры. Важно: для iOS требуется реализация кнопки явного запроса прав у пользователя на доступ к датчикам движения.
Математика: Использование сторонней JS-библиотеки (например, satellite.js) для расчета текущего Азимута и Элевации спутника на основе TLE-данных, текущего времени и GPS-координат смартфона.
Отрисовка: Наложение прозрачного слоя <canvas> поверх видео, на котором динамически (с частотой 60 fps) отрисовывается маркер спутника, смещающийся в зависимости от поворота телефона.
6. PWA и Оффлайн-режим
Приложение должно сохранять 100% работоспособность в местах без сотовой связи (в походе/лесу).
Service Worker: Скрипт, перехватывающий сетевые запросы.
Cache Storage: Используется для кэширования "каркаса" приложения (index.html, CSS, JS-файлы, иконки).
IndexedDB / LocalStorage: Используется для сохранения "мяса" — загруженного файла Frequencies.xml и последнего скачанного текстового файла TLE.
Установка на устройство: Наличие файла manifest.json с прописанными иконками, цветами и параметром display: standalone, позволяющим установить сайт как нативное приложение на экран "Домой" (iOS/Android) без адресной строки браузера.
Индикация состояния: UI-элемент, показывающий текущий статус (Online / Offline) и дату последней актуализации баз данных TLE и частот.
7. Механизм бесшовных обновлений (OTA)
При внесении изменений в код (новые кнопки, функционал) на GitHub:
Service Worker при наличии интернета в фоновом режиме обнаруживает изменение своей версии.
Новые файлы скачиваются в теневой кэш (не ломая текущую сессию пользователя).
Пользователю показывается ненавязчивый UI-баннер (Toast/Snackbar): "Доступна новая версия. [Обновить]".
При нажатии на кнопку вызывается skipWaiting() и принудительная перезагрузка страницы для применения обновления.
8. Инфраструктура и настройки на стороне GitHub
Для реализации проекта необходимо задействовать следующие механизмы GitHub:
Repository: Публичный репозиторий для хранения кода и XML.
GitHub Pages: Включить в настройках репозитория развертывание статического сайта из ветки main (или gh-pages).
GitHub Actions (CI/CD автоматизация): Написание небольшого YML-скрипта (воркфлоу), который будет запускаться по расписанию (cron) — например, раз в сутки. Скрипт должен обращаться к публичным API (Space-Track или CelesTrak), скачивать свежий текстовый файл TLE, и автоматически делать "Commit + Push" этого файла в репозиторий. Это обеспечит приложению всегда свежие данные орбит без нарушения политики CORS на стороне клиента.
